## 2. 产品总览页：平台闭环链路（建议放 1-2 张）

### 2.1 闭环链路一句话

- **认知**：竞赛资源库 + 技能图谱，让用户知道“要参加什么、需要什么技能”
- **行动**：孪孪伴学辅助学习/对话 + 学习路径规划，让用户知道“怎么学、按什么顺序学”
- **沉淀**：闪卡把学习结果固化成可复习的知识点 + 图谱化管理
- **检验**：闪卡测试生成试卷并批改，形成“练—测—改”的反馈回路
- **成长**：成长记录日历/里程碑/照片墙，把过程与成果留痕
- **职业**：职业展示看岗位与趋势；职业规划测评 + 工作流生成报告；简历制作 + AI润色；AI面试练习与复盘
- **回流**：职业目标反向驱动“技能/路径/闪卡”继续迭代

### 2.2 关键导航结构（来自 `NavBar.vue`）

- **首页**：`/home`
- **学习图谱（下拉）**：技能图谱 `/knowledge-graph`；学习路径图谱 `/learning-path-graph`；闪卡图谱 `/flashcard-graph`
- **孪孪伴学**：`/twin-study`
- **竞赛活动（下拉）**：竞赛一览 `/competition/edu`；学科专业竞赛 `/competition/subject`；竞赛详情 `/competition/detail/:id`
- **成长轨迹（下拉）**：成长记录 `/growth/record`；里程碑 `/growth/milestone`；照片墙 `/growth/photo-wall`
- **职业规划（下拉）**：职业展示 `/career/showcase`；职业规划 `/career/planning`；简历制作 `/career/resume`；AI 面试 `/career/ai-interview`

## 3. 系统入口与账号体系（建议放 1-2 张）

### 3.1 全局登录态与访问控制（`src/router/index.js`）

- **路由守卫**：除 `/login` 外均需登录（`localStorage.userInfo` 存在才放行）
- **已登录访问登录页**：自动跳转 `/home`
- **图谱页来源记录**：
  - 通过 `sessionStorage.graph_entry_origin` 记录“从首页/从孪孪伴学进入图谱”
  - 用于图谱页“返回”的一致体验

### 3.2 登录/注册（`LoginPage.vue`）

- **登录方式**：密码登录、短信验证码登录
- **注册**：账号 + 图片验证码 + 短信验证码 + 密码/确认密码（含强度与一致性校验）
- **图片验证码**：可刷新、并在流程中强制校验
- **忘记密码**：已有弹窗与校验 UI（页面提示后端接口文档未提供对应调用）

### 3.3 个人资料（`NavBar.vue` 弹窗）

- **编辑字段**：头像、昵称、性别、年级、专业、学校、个人简介
- **保存**：上传头像后再更新资料；并同步 `localStorage.userProfile`
- **退出登录**：清理本地登录信息并跳转 `/login`

## 4. 链路第一段：从“竞赛资源库”了解竞赛（建议 3-5 张）

### 4.1 竞赛资源库定位

- 让用户回答三个问题：
  - **有什么竞赛**：竞赛列表
  - **适合我吗**：学科分类/赛事级别/赛事阶段筛选
  - **怎么报名/在哪看**：竞赛详情 + 官网链接
- 这一步是闭环的起点：竞赛目标会反向驱动技能、学习路径、闪卡沉淀。

### 4.2 竞赛一览（`EduCompetitionPage.vue` / `/competition/edu`）

- **搜索**：按 `contestName` 防抖搜索（500ms）
- **筛选**：
  - 学科分类（分组下拉：文体/商科/综合/理科/工科 + 二级）
  - 赛事阶段：报名结束/比赛进行中/比赛结束
- **分页**：Element Plus 分页控件
- **跳转**：进入详情页 `/competition/detail/:id`
- **接口**：`competitionApi.getHonorContestList(params)` → `GET /contest/listHonor`

### 4.3 学科专业竞赛（`SubjectCompetitionPage.vue` / `/competition/subject`）

- **搜索**：按竞赛名防抖
- **筛选**：
  - 学科分类下拉 + 学科快捷标签（映射 classId）
  - 赛事级别：校级/市级/省级/全国性/全球性/自由/其他
  - 赛事阶段：报名结束/进行中/结束
- **接口**：`competitionApi.getContestList(params)` → `GET /contest/list`

### 4.4 竞赛详情（`ContestDetailPage.vue` / `/competition/detail/:id`）

- **加载/失败态**：loading、错误提示、返回
- **正文渲染**：展示后端返回 `content`（HTML）
- **外链**：`contestUrl` 存在时可跳转官网
- **接口**：`competitionApi.getContestDetail(contestId)` → `GET /contest/detail`

## 6. 链路第三段：孪孪伴学——学习对话 + 学习路径规划（建议 4-7 张）

### 6.1 孪孪伴学定位（`TwinStudyPage.vue` / `/twin-study`）

- 平台学习中枢，包含两类场景：
  - **默认对话**：学习问答（流式输出）
  - **学习路径对话**：生成学习路径 JSON（流式输出 + 可保存）

### 6.2 会话管理（左侧 Sidebar）

- 新建/切换/重命名/删除会话
- 会话元信息持久化在 `localStorage`（包含会话 id、title、更新时间等）

### 6.3 流式聊天（SSE）与内容解析

- **流式输出**：边接收边渲染（Markdown）
- **解析能力**：
  - `<think>...</think>` 思考块提取与显示控制
  - `<details><summary>outputList</summary>...</details>` 知识来源提取（支持 JSON 与 Python dict 兼容解析）
- **体验细节**：避免重复消息、可中止/结束流

### 6.4 学习路径规划（SSE）

- 通过 `learningPath.planStreamFlux(...)` 获取流式规划结果
- 接收 `meta` 事件保存 `sessionId`（用于润色/继续生成）
- 前端在流式过程中不断尝试“解析 JSON → 生成节点 → 画出思维导图式展示”

### 6.5 学习路径润色与保存

- **润色**：以现有路径 + 用户要求再发起规划流
- **保存**：弹出 `SavePathDialog` 并保存到“学习路径管理/学习路径图谱”

### 7.1 学习路径管理（`LearningPathManagePage.vue`）

### 7.2 学习路径图谱页（`LearningPathGraphPage.vue` / `/learning-path-graph`）

- **支持两种视角**：
  - 看某一条路径
  - 看所有路径合并视角（`LearningPathGraphAll`）

### 7.3 路径选择面板（`LearningPathSelectorPanel.vue`）

## 8. 链路第五段：闪卡——把学习结果沉淀为可复习的知识网络（建议 8-14 张）

### 8.1 闪卡体系：列表 + 图谱双模式

- **记忆闪卡（列表/复习）**：`FlashCardPage.vue`
- **闪卡图谱（暂存/图谱/复习/编辑/对比/归档）**：`FlashcardGraphPage.vue`

### 8.2 列表模式能力（`FlashCardPage.vue`）

- **加载**：`flashCardApi.list()`
- **复习列表**：`flashCardApi.reviewList()`
- **复习打分**：`flashCardApi.review(id, difficultyLevel)`
- **编辑**：`flashCardApi.update(payload)`
- **AI 辅助**：`flashCardApi.aiAssist(id, prompt)`
- **删除**：`flashCardApi.delete(id)`
- **实时通知**：
  - 动态加载 SockJS + STOMP
  - 订阅用户队列消息，收到 `flashcard_generated` 自动刷新

### 8.3 图谱页多视图状态机（`FlashcardGraphPage.vue`）

- **TEMP_ZONE**：暂存区（生成后待确认）
- **CATEGORY_SELECT**：层级分类选择（归档必经）
- **GRAPH**：Neo4j 闪卡图谱
- **REVIEW**：图谱内复习单卡
- **EDIT**：图谱内编辑单卡
- **COMPARE**：对比视图（与学习路径联动）

### 8.4 暂存区（TEMP_ZONE）：生成 → 等待 → 确认归档

- 暂存列表：`flashCardApi.getTempZoneList()`
- 生成进度：`flashCardApi.status(flashCardId)`（轮询）
- 过期天数：`flashCardApi.getTempCardExpiration(id)`
- 暂存删除：`flashCardApi.deleteTempCard(id)`
- 暂存详情：`flashCardApi.getTempCardDetail(id)`

### 8.5 归档（确认保存）：层级标签路径

- 分类树：`flashCardApi.getCategoryTree()`（若未启用，前端有默认分类树兜底）
- 确认保存：`flashCardApi.confirmSave({ id, hierarchyPath })`
- 更新标签：`flashCardApi.updateWithTags({ id, hierarchyPath, tags })`

### 8.6 闪卡图谱（GRAPH）：Neo4j 可视化与检索

- 图谱数据：
  - `flashCardApi.getGraphData({ userId })` → `GET /neo4j-graph`
  - 回退：`flashCardApi.list()`（当图谱不可用时）
- 搜索：
  - `flashCardApi.searchInNeo4j({ type:'ALL', keyword })`
  - 前端兜底本地匹配
- 时间筛选：
  - `flashCardApi.filterGraphByUpdated(range)` → `GET /flash-card/neo4j/graph-by-updated`
- Neo4j 同步更新/删除：
  - `flashCardApi.updateInNeo4j(payload)` → `POST /neo4j-graph/update`
  - `flashCardApi.deleteInNeo4j(payload)` → `POST /neo4j-graph/delete`

### 8.7 图谱内复习/编辑

- 点击节点进入单卡复习或编辑，操作完成后回到图谱
- 复习逻辑与列表复习一致（难度分级驱动下次复习时间）

### 8.8 对比模式（COMPARE）：学习路径 vs 闪卡对齐

- 由 `CompareView.vue` 承载对比 UI
- 主要用在“学习路径节点 ↔ 已沉淀闪卡知识点”的差距分析

## 12. 链路第九段：职业准备（中）——职业规划（测评 + 多 Agent 工作流演示）（建议 6-10 张）

### 12.1 页面结构（`CareerPlanningPage.vue`）

- **职业测评**：拉取题库 → 单选作答 → 生成测评结果 JSON
- **多 Agent 工作流演示**：把“用户输入 + 测评结果”送入工作流，流式展示进度，最终输出报告

### 12.2 职业测评（MBTI 风格）

- 题库接口：`GET /question/mbti`
- 作答方式：逐题单选、可回退
- 完成后生成 `assessmentResult`（JSON）供工作流使用

### 12.3 多 Agent 工作流演示（SSE + 最终报告轮询）

- 启动工作流：`POST /agent/career/start` → 返回 `runId`
- 订阅进度：`GET /agent/career/stream?runId=...`（SSE）
  - 前端实时更新一个 SVG 工作流图（节点状态变化：进行中/完成/失败）
- 获取工作流拓扑：`GET /agent/career/graph`
- 获取最终报告：`GET /agent/career/result?runId=...`
  - 返回 Markdown 报告 + 相关链接（前端展示并支持复制/跳转）

## 13. 链路第十段：简历制作——结构化输入 + 实时预览 + AI润色（建议 8-12 张）

### 13.1 简历制作页定位（`ResumeBuilderPage.vue` / `/career/resume`）

- 目标：让用户把“学习/项目/成长”沉淀为可投递的简历，并可一键导出 PDF
- 两栏布局：
  - 左侧：结构化表单输入
  - 右侧：A4 简历实时预览（可拖拽/可调样式）

### 13.2 简历数据结构（前端实际字段）

- `personalInfo`：姓名、电话、邮箱、意向岗位、个人主页等
- `education`：学校/专业/时间/学历等
- `projects`：项目名称、角色、时间、技术栈、亮点（支持 Markdown）
- `workExperience`：公司、职位、时间、职责与成果（支持 Markdown）
- `skills`：技能列表（可分组）
- `awards`：奖项/证书/竞赛成果等

### 13.3 AI 润色（`chatApi.polishText`）

- 每个描述型 textarea 旁提供“机器人按钮”
- 点击后把当前文本发送给后端润色接口并回填

### 13.4 预览样式与可配置项

- 主题色 `themeColor`
- 字号 `fontSize`
- 行高 `lineHeight`
- 段间距 `sectionSpacing`
- 页边距 `margin`
- 预览区支持拖拽定位（便于对齐/查看）

### 13.5 数据导入导出与持久化

- **预览填充**：一键填充示例数据（便于用户快速上手）
- **清空**：清空当前数据
- **导出 JSON**：下载简历数据 JSON
- **导入 JSON**：上传 JSON 恢复简历
- **持久化**：
  - `localStorage.resumeData`
  - `localStorage.resumeSettings`

### 13.6 导出 PDF

- 点击“导出 PDF”调用 `window.print()`（依赖浏览器打印为 PDF 的能力）

## 14. 链路第十一段：AI 面试——简历接入→分析→面试会话→复盘报告→历史（建议 10-16 张）

### 14.1 AI 面试总览页（`AIInterviewPage.vue` / `/career/ai-interview`）

页面以 4 阶段呈现一条完整流程：

1) **简历接入（Resume Intake）**  
2) **简历分析（Analysis）**  
3) **面试会话（Interview Session）**  
4) **复盘评估（Review）**

并提供历史记录抽屉（History Drawer）。

### 14.2 简历接入：平台简历 / 上传简历 两种来源

- **来源类型**：`resumeSource = PLATFORM | UPLOAD`
- 平台简历入口：引导跳转到 `/career/resume` 编辑
- 上传简历：调用 `aiInterviewApi.uploadResume(file)`
- 已上传简历列表：`aiInterviewApi.listResumes()`

### 14.3 简历分析

- 选择某份简历后发起分析：`aiInterviewApi.analyzeResume(resumeId, payload)`
- 分析结果写入页面状态 `analysisResult`
- 面试阶段依赖分析结果与职位方向等参数

### 14.4 创建面试会话与进入会话页

- 创建会话：`aiInterviewApi.createSession(payload)` → 返回 `sessionId`
- 跳转到会话页：`/career/ai-interview/session?sessionId=...`

### 14.5 会话页：三种会话实现分支（`AIInterviewSessionPage.vue`）

根据后端/配置返回的 mode 渲染不同组件：

- **POLLING**：`PollingInterviewSession`（轮询下一题/提交答案）
- **REALTIME**：`RealtimeInterviewSession`（具备实时能力，含 WebSocket 交互）
- **default**：`InterviewSession`（通用实现兜底）

### 14.6 面试过程中的关键接口能力（`aiInterviewApi.js`）

- 下一题：`aiInterviewApi.nextQuestion(sessionId, payload)`
- 上传语音答案：`aiInterviewApi.uploadAnswerAudio(sessionId, file)`
- 结束会话：`aiInterviewApi.finishSession(sessionId)`
- 获取报告：`aiInterviewApi.getReport(reportId)` 或按 session 获取
- 报告列表：`aiInterviewApi.listReports()`
- 删除报告：`aiInterviewApi.deleteReport(reportId)`
- 清空历史：`aiInterviewApi.clearReports()`

### 14.7 面试结束→报告结构转换→复盘闭环

- 会话结束后会将 `reportJson` 映射为前端可展示的 review 结构
- 并进行两类持久化：
  - `sessionStorage.ai-interview-pending-review`：用于回到主页面继续“复盘阶段”
  - `localStorage.aiInterviewHistory`：保存历史记录，供历史抽屉复看

### 14.8 阶段状态的本地保存（保证中途刷新不丢）

- `sessionStorage.ai-interview-resume-state`：保存简历来源/选择状态
- `sessionStorage.ai-interview-pending-review`：保存待复盘报告

## 15. 技术实现与关键数据流（前端视角）（建议 6-10 张）

### 15.1 技术栈总览（来自 `package.json` 与项目结构）

- **框架**：Vue 3 + Vite
- **UI**：Element Plus
- **图谱可视化**：
  - D3（部分图组件）
  - 3d-force-graph / Three.js（Neo4j 图谱渲染支撑）
- **网络通信**：
  - Axios（普通 REST）
  - `@microsoft/fetch-event-source`（SSE 流式）
- **实时通信**：SockJS + STOMP（闪卡生成通知）
- **编辑与渲染**：
  - Monaco Editor（闪卡编程题编辑）
  - Markdown 渲染（marked/markdown-it）
  - DOMPurify（HTML 消毒，防 XSS）

### 15.2 三类“实时/准实时”交互模式对比

- **SSE（流式文本）**：
  - 孪孪伴学默认对话（stream chat）
  - 学习路径规划（plan stream）
  - 职业规划工作流（career stream）
- **WebSocket（推送通知）**：
  - 闪卡生成完成事件 `flashcard_generated`
- **轮询（状态查询）**：
  - 暂存闪卡生成进度（每 5 秒）
  - AI 面试（POLLING 模式的下一题/状态）

### 15.3 图谱页面统一的“iframe 通讯范式”

- 技能图谱/闪卡图谱/学习路径图谱都存在“图谱渲染层与业务 UI 层分离”的设计
- 业务层通过 postMessage 控制搜索、选择节点、接收选中结果

### 15.4 本地存储设计（闭环数据的“前端记忆”）

- `localStorage`：
  - `userInfo`：登录态
  - `userProfile`：头像/昵称等
  - `resumeData` / `resumeSettings`：简历与样式
  - `aiInterviewHistory`：AI 面试历史
  - 孪孪伴学会话元信息
  - 闪卡图谱最后视图 `flashcard_last_view`
  - 成长轨迹侧边球位置（成长页面专用）
- `sessionStorage`：
  - `graph_entry_origin`：图谱页返回来源
  - 学习路径规划 sessionId（用于继续/润色）
  - AI 面试阶段状态与待复盘数据

### 15.5 典型数据流：以“学习路径→闪卡→测试→简历→AI面试”为例

- 学习路径：孪孪伴学流式生成 → 用户确认保存 → 学习路径图谱展示
- 闪卡沉淀：针对路径节点生成/整理闪卡 → 暂存区确认归档 → 进入闪卡图谱
- 测试反馈：基于节点/路径生成试卷 → 作答 → AI 批改 → 逐题解析 → 发现薄弱点 → 回到孪孪伴学/闪卡补强
- 简历输出：把项目/学习成果/竞赛奖项写进简历 → AI 润色 → 导出 PDF
- AI 面试：选择简历 → 分析 → 面试会话 → 报告复盘 → 基于反馈回流到“技能/路径/闪卡”
